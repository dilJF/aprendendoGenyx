Get started

After download template extract package and view the contain in html directory, dir should look like this

First start with copy the css, js and images folders to your new project dir, copy the blank.html file too this is your start file where you need to add contain.

It`s time to observe other directories:

Directories structure

CSS directory

Directory look like this:

 

in folder bootstrap is stored based bootstrap.css file

in folder fonts are font-face icons from icomoon 

in folder genyx-theme is stored jquery theme created for this template

app.css - contain all important styles for this template

custom.css is empty file here you need to put your own changes 

Please use this file not app.css because if i release some updates you will lose your changes

icons.css - contain styles for icons , open file and remove line 20 - 23, this is only for demo. Leater i will explain how to add icons in template.

 

Js (javascript) directory

Content of directory:

bootstrap - contain core js funtion for bootstrap framework

pages - directory contain all javascript init functions for every page from demo ( use domready.js as starting point for new file )

plugins - This directory contain every plugin used by template , seperated by meaning and functions

Every plugin is need to be included and initialized in order to work, plugins are sepereated for easy managment in development process , in production site you may be wanted to merge files for small http queries.

Please look Plugins section for more information.

File app.js is the core javascript file this file need to be included in every page, here is initialize the special made jquery plugin for this template jquery.genyxAdmin.js Here is the full list of options to control the template.


	$.genyxAdmin({



	        fixedWidth: false,//put true if you want fixed version



	        customScroll: false,// Custom scroll for page. true or false 



	        responsiveTablesCustomScroll: false,// custom scroll for responsive tables. true or false



	        backToTop: true,//show back to top , true or false



	        navigation: {



	            useNavMore: true, //use arrow for hint. ture or false



	            navMoreIconDown: 'i-arrow-down-2', //icon for down state.



	            navMoreIconUp: 'i-arrow-up-2',//icon for up state



	            rotateIcon: true//rotate icon on hover , true or false



	        },



	        setCurrent: {



	            absoluteUrl: false, //put true if use absolute path links. example http://www.host.com/dashboard instead of /dashboard



	            subDir: '' //if you put template in sub dir you need to fill here. example '/html'



	        },



	        collapseNavIcon: 'i-arrow-left-7', //icon for collapse navigation button



	        collapseNavRestoreIcon: 'i-arrow-right-8', //icon for restore navigation button



	        rememberNavState: true, //remember if menu is collapsed 



	        remeberExpandedSub: false, //remeber expanded sub menu by user



	        hoverDropDown: true, //set false if not want to show dropdown on hover ( click instead)



	        accordionIconShow: 'i-arrow-down-2',//icon for accordion expand



	        accordionIconHide: 'i-arrow-up-2'//icon for accordion hide





	       showThemer: true//put false to remove themers



	    });

Every options is well commented but if have problems just drop me a line.

So let`s say you want to hide loader in the begining of page so you need to put false instead of true.

$.genyxAdmin({preloaderAnimation: false});

no need to put other options because this is  by default.

widget get the width of parrent element so in almost every case you will use it with bootstrap grid system, i`m not try to explain this system because is well documented in official site so if you not familar with bootstrap please read the documentation here:

Please get familar with basic bootstrap functions here: http://twitter.github.io/bootstrap/scaffolding.html#gridSystem

Widget have two looks one is the defaulth another is the plain to use plain widget you need to add class plain after class widget

<div class="widget plain"></div>

Some times you mau not want widget to be collapsible in this case just remove the minimize button

<a href="#" class="minimize"></a>

Other times you may need widget to be closed by defaulth so in this case you need to add class closed

<div class="widget closed"></div>

Widget content has 15px padding so if you want to remove it add class noPadding to content

<div class="widget-content noPadding">

Usage

 
Simple add i_{iconname} to an element:
 
<i class="i_home"></i>
This will use the icon with the name "home".
 
Mostly used sizes are predefined with this short code
 

	.icon12 {font-size: 12px;}



	.icon16 {font-size: 16px;}



	.icon20 {font-size: 20px;}



	.icon24 {font-size: 24px;}



	.icon32 {font-size: 32px;}



	.icon64 {font-size: 64px;}
So if you want to show home icon whit size 24px just write this code
 
<i class="icon24 i-home></i>
 
You may have to add additional styling information on that element to display the icon properly

Jrespond

jRespond is a simple way to globally manage JavaScript on responsive websites.
 
Responsive websites that require JavaScript functionality for some breakpoints and not for others need some type of system for triggering the correct funcitons at the correct breakpoint and to also be aware of when a browser is resized across breakpoints. Although switching between breakpoints could be seen as an edge case, a few applications for jRespond are:
 
Managing functionality for initial page load: Even if the browser is never resized, jRespond can help manage what JavaScript functions are triggered when the page loads for a given breakpoint.
Development testing: jRespond makes it much easier to test in-browser.
Borderline device widths: Real user browser resizing and device rotation that crosses breakpoints.
 
Plugin is located in:
js/plugins/core/jrespond
 
Plugin need to be loaded like this
<script src="js/plugins/core/jrespond/jRespond.min.js"></script>
Official documentattion is here: https://github.com/ten1seven/jRespond

his jQuery plugin supplies an easy way to unobtrusively add a letter-based navigation widget to any UL or OL list. An easily stylable (via CSS) nav bar appears above the list, showing the user the letters A-through-Z. Clicking one of the letters filters the list to show only the items in the list that start with that letter.
